Dica Extra: Versionamento com Git e GitHub via NetBeans

Envio do projeto da atividade da academia para o repositório GitHub, utilizando o NetBeans.

Etapa 1 – Compreendendo o problema

🔎 O que é versionamento?

Versionamento é o controle das versões de um projeto de software ao longo do tempo. Isso permite:

  • Salvar o progresso com segurança
  • Voltar para versões anteriores, se necessário
  • Trabalhar em colaboração com outras pessoas

O Git é um sistema de versionamento local e o GitHub é a plataforma que hospeda os repositórios online (nuvem).

💡 O que vamos fazer?

Você vai:

  • Criar um repositório no GitHub
  • Enviar (fazer commit e push) o projeto da atividade da disciplina
  • Aprender a manter o projeto atualizado:
    • 💻 Usando a mesma máquina (Situação 1)
    • 🏠🧑‍🏫 Alternando entre casa e computador da instituição (Situação 2)

🛠️ Etapa 2 – Implementação guiada

Passo 1 – Criar o repositório no GitHub

  1. Acesse https://github.com
  2. Clique em New Repository
  3. Preencha:
    • Nome: projeto-disciplina (ou outro nome significativo)
    • Descrição: Projeto da disciplina XX – IFSul
    • Deixe vazio, sem README ou .gitignore
    • Visibility: Public (pois para avaliação das atividades eu devo ter acesso ao seu projeto)
  4. Clique em Create repository

Passo 2 – Vincular o projeto no NetBeans

2.1 Inicializando o Git localmente

  1. Abra o NetBeans
  2. Selecione o projeto que deseja enviar para o repositório no Github
  3. Com o botão direito sob o projeto clique em Versionamento > Inicializar Repositório Git
  4. Confirme a pasta do projeto e clique em OK

Isso cria um repositório Git local no seu projeto

2.2 Realizando o primeiro commit

  1. Clique com o botão direito no projeto novamente
  2. Vá em Git > Commit
  3. Marque todos os arquivos listados (use o botão “Selecionar Tudo”)
  4. Escreva uma mensagem como:
    “Versão inicial do projeto XXX”
  5. Clique em Commit

2.3. Vincular repositório remoto no NetBeans

  1. No NetBeans, clique com o botão direito no projeto
  2. Vá em Git > Remotes > Push…
  3. Seleciona a opção Especificar Localização do Repositório Git
  4. Configure os campos da seguinte forma:
  • Nome do Remoto: mantenha origin
  • Marque a opção Persistência Remota para guardar os dados (por exemplo, se desejar evitar digitar o token toda vez)
  1. No campo URL, cole o link do seu repositório no GitHub (exemplo: https://github.com/seu-usuario/projeto-disciplina.git)
  2. Insira suas credenciais do GitHub
  • Nome de usuário
  • Token de acesso pessoal no lugar da senha

O GitHub não aceita mais senha como login em operações de push/pull. Para isso, você precisa gerar um token de acesso e usá-lo como se fosse a senha. Se você não possui um token criado e guardado para utilização siga o próximo passo.

  1. Clique em Avançar (Next) e depois Concluir (Finish)

2.4 Criação de Token no Github

Se você ainda não possui um token de acesso, siga os passos abaixo para criar um novo:

  1. Acesse seu perfil no GitHub:
    🔗 https://github.com
  2. No canto superior direito, clique na sua foto e vá em
    Settings > Developer Settings
  3. Clique em Personal access tokens > Tokens (classic)
  4. Clique no botão Generate new token (classic)

Configure o token:

  • Note (nome do token): exemplo Token para NetBeans
  • Expiration (data de validade): você pode customizar para que o token tenha validade, pelo menos, até o final do semestre.
  • Scopes (Permissões): marque apenas:
    • repo (permite acesso e edição de repositórios)
  • Role até o fim da página e clique em Generate token

Importante:

  • Copie o token assim que ele for gerado
  • Guarde em local seguro — você não poderá visualizar novamente
  • Esse token será usado no campo “senha” do NetBeans em futuras operações com GitHub

2.5 Enviar os arquivos (Push)

  1. Após vincular o repositório remoto, vá em Git > Remotes > Push…
  2. Confirme que o branch é main
  3. Clique em Next, depois em Finish

Pronto! Agora seu projeto foi enviado para o GitHub

Como saber se deu certo?

Acesse o repositório no GitHub (navegador) e verifique se os arquivos do projeto aparecem na interface.

🔁 Como trabalhar com Git em diferentes cenários

💻 Cenário 1 – Usando a mesma máquina (ex: notebook pessoal)

Sempre que fizer uma alteração deverá fazer a ação Commit & Push:

  1. Realize a ação commit - ver item 2.2
  2. Após comitado os dados, faça a ação push - repita os passos descritos no item 2.5

Isso envia a nova versão para o GitHub e mantém tudo sincronizado.

🏠🧑‍🏫 Cenário 2 – Trabalhando em casa e no laboratório do IFSul com máquinas diferentes

Sempre que for mudar de máquina:

🏠 No computador de casa:

  1. Faça alterações normalmente
  2. Realize a ação Commit & Push ao finalizar:
  • Realize a ação commit - ver item 2.2
  • Após comitado os dados, faça a ação push - repita os passos descritos no item 2.5

Isso envia a nova versão para o GitHub e mantém tudo sincronizado.após finalizar

🧑‍🏫 No laboratório do IFSul (nova máquina):

  1. Abra o NetBeans
  2. Vá em Team > Git > Clone
  3. Cole o link do seu repositório
  4. Faça o clone em uma pasta nova
  5. Trabalhe normalmente
  6. Ao final, realize a ação Commit & Push para atualizar o repositório
  • Realize a ação commit - ver item 2.2
  • Após comitado os dados, faça a ação push - repita os passos descritos no item 2.5

⚠️ Se já tiver o projeto clonado no laboratório:

Antes de começar a trabalhar:

  • Vá em Git > Remote > Fetch
  • Depois em Git > Merge > Remote Tracking
  • Assim, você baixa as alterações feitas em casa
