Aula 2 ( parte 2 )

Relacionamento entre Objetos (composição de coleção)

Tema: Armazenando múltiplos objetos em uma classe (Composição com coleção)

Na modelagem orientada a objetos, é comum que um objeto esteja relacionado a vários outros objetos. Esse tipo de relação é conhecido como relacionamento de um para muitos (1:N).

Exemplo: Pessoa e AvaliacaoFisica

No exemplo abaixo, temos duas classes:

  • Pessoa: representa uma pessoa com nome, data de nascimento e CPF.

  • AvaliacaoFisica: representa uma avaliação com dados de saúde.

A avaliação física está relacionada a uma única pessoa, ou seja, no contexto fictício de Acadêmia cada aluno deve passar períodicamente por uma avaliação física, que a partir disso poderá gerar novas prescrições de treinos.

O diagrama a seguir representa o relacionamento entre as classes Pessoa e AvaliacaoFisica. Cada Pessoa pode ter zero ou mais avaliações físicas associadas, caracterizando uma composição 1:N.

Explicação do Relacionamento

Explicação

A classe Pessoa contém:

  • Dados pessoais (nome, dataNascimento, CPF)

  • Uma lista de avaliações físicas (List)

  • Métodos para adicionar e exibir avaliações

A classe AvaliacaoFisica:

  • Está associada a uma única Pessoa (aluno)

  • Possui atributos para data, peso, altura, idade e IMC

  • Contém métodos para calcular IMC e idade com base na data de nascimento do aluno

Conclusão

O relacionamento de composição com coleção é essencial para modelar situações onde um objeto precisa armazenar múltiplas instâncias relacionadas a ele.

Neste exemplo, a classe Pessoa representa um aluno, e cada avaliação física realizada é representada por uma instância da classe AvaliacaoFisica. Com isso, conseguimos manter um histórico completo das avaliações dentro da própria classe Pessoa.

Através de métodos específicos (como adicionarAvaliacao() e getAvaliacoes()), podemos organizar, acessar e manipular esse conjunto de objetos com segurança e clareza.

Atividade Prática – Lista de Avaliações

Implemente a relação entre AvaliacaoFisica e Pessoa a fim de permitir manter o histórico de avaliações realizadas sobre um determinado aluno.

  1. Na classe Pessoa adicione o atributo:

    • List<AvaliacaoFisica> avaliacoes com o devido encapsulamento (atributo private) e crie o método getAvaliacoes() para retornar a lista.

  2. Nos construtores da classe, inicialize a lista:

this.avaliacoes = new ArrayList<>();

  1. Crie o método adicionarAvaliacao(AvaliacaoFisica avaliacao) de modo que permita adicionar uma avaliação a lista.

  2. Ajuste o método exibirDados() para mostrar também a quantidade de avaliações.

  3. Teste a implementação na classe Teste

  • Crie um aluno
  • Adicione 2 avaliações físicas para ele
  • Mostre o método exibirDados() do aluno cadastrado

Exemplo de Teste

Pessoa p1 = new Pessoa("Juka");

AvaliacaoFisica a1 = new AvaliacaoFisica(...);
AvaliacaoFisica a2 = new AvaliacaoFisica(...);

p1.addAvaliacao(a1);
p1.addAvaliacao(a2);

System.out.println(p1.exibirDados());
