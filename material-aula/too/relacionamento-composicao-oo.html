Orientação a Objetos

Aula 2 - Relacionamento entre Objetos (Composição)

Na programação orientada a objetos, relacionar classes é uma forma natural de modelar o mundo real, onde objetos frequentemente dependem ou contêm outros objetos.

Uma classe pode ter atributos que são instâncias de outras classes, estabelecendo um relacionamento entre elas.

Exemplo: Pessoa e AvaliacaoFisica

No exemplo abaixo, temos duas classes:

  • Pessoa: representa uma pessoa com nome, data de nascimento e CPF.

  • AvaliacaoFisica: representa uma avaliação com dados de saúde.

A avaliação física está relacionada a uma única pessoa, ou seja, no contexto fictício de Acadêmia cada aluno deve passar períodicamente por uma avaliação física, que a partir disso poderá gerar novas prescrições de treinos.

Explicação do Relacionamento

Neste caso, a classe AvaliacaoFisica possui um atributo do tipo Pessoa, representando o aluno ao qual a avaliação pertence:

private Pessoa aluno;

Isso representa uma relação de “possui”:

“Uma avaliação física pertence a uma pessoa (a um aluno)”.

Essa relação pode ser chamada de:

  • Composição – porque a AvaliacaoFisica não faz sentido existir sem estar vinculada a uma Pessoa.

  • Representa uma relação forte entre objetos.

  • Quando a AvaliacaoFisica é destruída, o vínculo com Pessoa também se encerra no contexto da avaliação.

Logo, para modelagem da Avaliação Física, a classe AvaliacaoFisica deve possuir um objeto Pessoa como atributo, referenciando a quem corresponde a avaliação realizada.

Conclusão

  • Modelar uma classe com outra classe como atributo permite representar a realidade com mais precisão.
  • O relacionamento entre objetos torna o código mais modular, reutilizável e fácil de manter.
  • A composição é uma forma poderosa de organizar os dados e comportamentos do sistema, estabelecendo relações naturais entre os objetos.

Atividade – Classe AvaliacaoFisica

Implemente a classe AvaliacaoFisica em Java, associando-a a uma instância da classe Pessoa, representando o aluno avaliado. A atividade visa exercitar os conceitos de encapsulamento, relacionamento entre objetos, e a manipulação de datas e métodos de instância.

Requisitos de Implementação

  1. Encapsular todos os atributos da classe AvaliacaoFisica

    • Use private para os atributos.
    • Crie métodos get e set para os atributos conforme julgar necessário.

  2. Associar um aluno via construtor

    • O objeto Pessoa (aluno) deve ser passado como parâmetro no momento da criação de uma AvaliacaoFisica.
    • Exemplo: 
      Pessoa p1 = new Pessoa("Vivente");
AvaliacaoFisica a1 = new AvaliacaoFisica(p1);

  3. Inicializar a data da avaliação com a data atual no próprio construtor

  4. Ajuste o encapsulamento a fim de:

    • permitir alteração posterior da data da avaliação
    • Não permitir alteração do aluno após a criação, apenas a visualização

  5. Implementar o método calcularIMC()

    • Fórmula: peso / (altura * altura)
    • Retornar um double.

  6. Implementar o método calcularIdade()

    • Calcular a idade com base na dataNascimento da Pessoa.
    • Atualizar o atributo idade na AvaliacaoFisica.

  7. Criar o método exibirDados()

    • Deve mostrar:
      • Nome do aluno
      • Idade
      • Data da avaliação (formatada)
      • IMC calculado
