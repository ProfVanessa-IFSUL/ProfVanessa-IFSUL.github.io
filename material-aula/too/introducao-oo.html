Orientação a Objetos

Aula 1 - Introdução

A Programação Orientada a Objetos (POO) é um paradigma de programação que modela o software com base em objetos do mundo real, unindo dados (atributos) e comportamentos (métodos) dentro de estruturas chamadas classes.

A linguagem Java foi projetada desde sua origem com suporte nativo à orientação a objetos, sendo um dos exemplos mais utilizados para ensino e aplicação desse paradigma.

Cada objeto é uma instância de uma classe, que define um tipo de dado abstrato. A OO promove os princípios de encapsulamento, herança, polimorfismo e abstração como seus pilares fundamentais.

Conceitos Fundamentais da Orientação a Objetos

Conceito Descrição Exemplo em Java
Classe Molde que define a estrutura e comportamento de um objeto. class Pessoa { String nome; void falar() {} }
Objeto Instância de uma classe, contendo estado e comportamento. Pessoa p = new Pessoa();
Encapsulamento Oculta os detalhes internos e expõe apenas o necessário por meio de métodos públicos. definição de visibilidade dos atributos como private e acesso pelso métodos get e set.

Definindo o modelo de um objeto

Por meio da criação de classes conseguimos definir o modelo de determinado objeto. Por exemplo, em um cenário de uma academia podemos ter as pessoas, sendo aluno ou professor, a avaliação física e a prescrição do treino. Nesse cenário podemos modelar a Pessoa (podendo ser Aluno ou Professor), AvaliacaoFisica, e a PrescricaoTreino. Nesse momento não vamos considerar a especialização de pessoa como Aluno ou Professor, considerando todos apenas como Pessoas, pois esse conteúdo de herança veremos mais a frente. Considerando ainda que toda AvaliaçãoFísica é referente a uma Pessoa, e toda PrescricaoTreino é gerado após um uma AvaliacaoFisica, estando vinculado a uma avaliação.

Primeiro, vamos considerar a declaração da classe Pessoa, considerando que a Pessoa possui informações de nome, data de nascimento, e CPF.

Classe Pessoa – Definindo um Modelo de Objeto

Encapsulamento

Encapsulamento é um dos pilares da programação orientada a objetos. Ele consiste em

  1. Ocultar os atributos internos de uma classe, tornando-os inacessíveis diretamente de fora da classe, ou seja, privando o acesso por meio da definição privada nos atributos, como em: private String nome;
  2. Permitir o acesso controlado a esses atributos por meio de métodos públicos chamados de getters e setters
public String getNome() {
	return nome;
}
public void setNome(String nome) {
	this.nome = nome;
}

Explicação

  • private → impede o acesso direto aos atributos fora da classe.
  • getNome() → retorna o valor do atributo nome.
  • setNome(String nome) → permite alterar o valor de nome, de forma segura.

Como Criar Getters e Setters

  • O nome do método começa com get ou set.
  • A primeira letra do nome do atributo é colocada em maiúscula no nome do método.
  • O get retorna o tipo do atributo.
  • O set recebe o novo valor como parâmetro.

Exemplo Prático: Usando Get e Set

public class TestePessoa {
    public static void main(String[] args) {
        Pessoa p = new Pessoa("Fulano da Silva");

        // Usando os métodos para acessar
        p.setCpf("111.222.333-44");

        System.out.println("Nome: " + p.getNome());
        System.out.println("CPF: " + p.getCpf());
    }
}

Benefícios do Encapsulamento

Benefício Explicação
Segurança Impede que valores inválidos sejam atribuídos diretamente aos atributos.
Organização Torna o código mais modular, limpo e de fácil manutenção.
Controle Permite aplicar regras de validação dentro dos métodos set.
Flexibilidade Facilita futuras mudanças na estrutura da classe sem afetar outras partes do sistema.

Método exibirDados()

Nesse código o método exibirDados() tem como objetivo retornar uma String formatada contendo os dados da pessoa, organizados de forma legível, incluindo o nome, CPF (se disponível) e a data de nascimento (se disponível), formatada no padrão brasileiro (dd/MM/yyyy).

Esse método não recebe parâmetros, pois utiliza os atributos da própria instância da classe (nome, CPF, dataNascimento).
